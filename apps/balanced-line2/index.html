Balanced-line simulation

Based loosely on the dice game described in Eli Goldratt's "The Goal", this version can compare a "perfect" line (no variation in processes) with the more realistic case of random breakdowns or delays in processes.

The yellow boxes represent operations, with the count (in blue) being the number of units that were processed in the current ticki>. The white boxes represent the number of items available to be processed. The actual capacity of an operation to process units may be higher than shown, but is limited by the number of available units.

The cyan boxes (to the right of each line) represent cash, which starts at zero. Each unit costs $1, incurred when that unit first enters the line. Each finished unit earns $2. The simulation runs for 300 ticks, after which we deduct the running costs of $800. If the cash is below zero at the end of the run, the line can't pay its bills.

Balanced line, zero variation ("perfect")

Each operation is able to process exactly 3 units each 'ticks'. Moving of items between operations is assumed to be instantaneous.

Balanced line, small random variation

Each operation is able to process from 2 to 4 units per tick, with a mean of 3.

Balanced line, large random variation

Each operation is able to process from 0 to 7 units per tick, with a mean of 3.

Prediction is that higher variation will result in less throughput per time, and more WIP, even though the average capacity of all operations is the same.
